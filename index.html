📊 Approximation Analysis (CLO‑3)

▾

Definition: An algorithm has approximation ratio ρ(n) if for all inputs, |solution| ≤ ρ(n)·|OPT| (for maximization).

Greedy for Subset Sum: Has NO constant approximation ratio. Counterexample: S={⌊t/2⌋+1, ⌊t/2⌋+1, 1,1,...,1} — greedy picks large numbers and fails. Greedy can be arbitrarily bad.

DP: Solves decision problem optimally in O(n·t) — pseudo‑polynomial, not an approximation algorithm.

First‑fit/Brute-force: No guarantee (brute-force is exact but exponential).

📐 Formal NP‑Completeness Proof (CLO‑5)

▾

1. NP Membership: Given certificate C ⊆ S, verify ∑C = t in O(|C|) ≤ O(n) time. ✓ Subset Sum ∈ NP.

2. Reduction: 3‑SAT ≤p Subset Sum
Given 3‑CNF φ with n variables x₁...xₙ and m clauses C₁...Cₘ, construct n+m digit numbers (base 10):
• For each variable xᵢ: create vᵢ (xᵢ=true) and vᵢ' (xᵢ=false). Digit i = 1. For each clause Cⱼ containing xᵢ, digit n+j += 1 in vᵢ. For each clause containing ¬xᵢ, digit n+j += 1 in vᵢ'.
• For each clause Cⱼ: add sⱼ (1 in digit n+j) and sⱼ' (1 in digit n+j) — "slack" numbers.
Target t: digit i = 1 for i=1..n, digit n+j = 3 for j=1..m.

3. Correctness (⇒): If φ satisfiable, choose vᵢ for TRUE, vᵢ' for FALSE. Each clause satisfied → at least one TRUE literal → clause digit sum ≥1. Add slacks to reach exactly 3. Total sum matches t.

4. Correctness (⇐): If subset sums to t, each variable digit sum =1 → exactly one of vᵢ,vᵢ' chosen. Clause digits sum to 3 → at least one TRUE literal → all clauses satisfied.

5. Polynomiality: Reduction constructs 2n+2m numbers, each with n+m digits. Construction time O((n+m)²).

6. Conclusion: Since 3‑SAT is NP‑complete and 3‑SAT ≤p Subset Sum, Subset Sum is NP‑complete.

📊 Growth Analysis: 2ⁿ vs n·t (Pseudo‑polynomial)

DP runs in O(n·t) — polynomial in t's value, not bits. If t ≈ 2ⁿ, DP becomes exponential. This does not contradict NP‑completeness.
